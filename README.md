# Telegram-бот для мониторинга торрент-раздач по RSS

Бот следит за RSS-лентами торрент-трекеров (rutracker, kinozal и любыми другими,
у которых есть RSS) и присылает уведомление, как только появляется новая раздача.
В уведомлении: название, категория, качество (разрешение/источник/звук, если
удалось распознать из заголовка), размер, сиды/пиры (если есть в описании фида),
дата и ссылка.

## 1. Получить токен бота

1. Откройте Telegram, напишите **@BotFather**.
2. Команда `/newbot`, следуйте инструкциям.
3. Получите токен вида `123456789:AA...` — он понадобится дальше.

## 2. Установка на VPS (Ubuntu/Debian)

```bash
# Устанавливаем Python и venv, если ещё нет
sudo apt update
sudo apt install -y python3 python3-venv git

# Копируем проект на сервер (например, через scp или git)
sudo mkdir -p /opt/torrent-rss-bot
sudo chown $USER:$USER /opt/torrent-rss-bot
cd /opt/torrent-rss-bot
# ... скопируйте сюда все файлы проекта ...

# Создаём виртуальное окружение и ставим зависимости
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Настраиваем переменные окружения
cp .env.example .env
nano .env   # впишите BOT_TOKEN, при желании поменяйте остальные настройки
```

## 3. Проверка запуска вручную

```bash
source venv/bin/activate
python bot.py
```

Если всё настроено верно, в логе появится сообщение о старте фонового цикла
опроса, а бот начнёт отвечать в Telegram. Остановить — `Ctrl+C`.

## 4. Автозапуск через systemd (рекомендуется)

```bash
# Создаём отдельного пользователя для бота (не обязательно, но полезно)
sudo useradd -r -s /bin/false botuser
sudo chown -R botuser:botuser /opt/torrent-rss-bot

# Копируем unit-файл
sudo cp torrent-rss-bot.service /etc/systemd/system/

# Если меняли путь /opt/torrent-rss-bot или имя пользователя —
# поправьте это в /etc/systemd/system/torrent-rss-bot.service

sudo systemctl daemon-reload
sudo systemctl enable --now torrent-rss-bot

# Проверка статуса и логов
sudo systemctl status torrent-rss-bot
sudo journalctl -u torrent-rss-bot -f
```

Теперь бот будет работать в фоне и автоматически перезапускаться при сбое
или перезагрузке сервера.

## 5. Как пользоваться ботом

Напишите боту в Telegram:

- `/add <ссылка> [название]` — добавить RSS-фид. Название необязательно,
  если не указать — возьмётся заголовок самого фида.
  Пример: `/add https://rutracker.org/forum/rss.php?filter=... Новинки кино`
- `/list` — список ваших фидов с их id и статусом.
- `/remove <id>` — удалить фид (id смотрите в /list).
- `/interval <минуты>` — задать, как часто проверять ваши фиды (по умолчанию
  15 минут, минимум 3).
- `/check` — проверить все фиды прямо сейчас, не дожидаясь таймера.
- `/status` — сводка: сколько фидов, есть ли ошибки.

**Важно:** при добавлении фида бот запоминает все текущие раздачи в нём как
уже "виденные" и не заваливает вас историей — уведомления приходят только
о том, что появится в фиде *после* добавления.

## Где взять RSS-ссылку трекера

На большинстве трекеров RSS генерируется на странице результатов поиска —
обычно там есть иконка RSS или пункт меню «RSS» рядом с фильтрами поиска.
Для некоторых трекеров (например, rutracker) RSS по конкретным разделам/тегам
доступен только авторизованным пользователям — тогда в ссылку нужно
подставить ваш персональный RSS-ключ, который трекер выдаёт в настройках
профиля.

## Технические детали

- **Стек:** Python 3.11+, aiogram 3.x, aiohttp, feedparser, aiosqlite.
- **Хранилище:** SQLite (`bot.db` рядом с ботом), никаких внешних БД не требуется.
- **Многопользовательность:** каждый чат — свой набор фидов и свой интервал
  проверки, база это разделяет по `chat_id`.
- **Анти-флуд:** между отправкой уведомлений внутри одного цикла — пауза,
  плюс корректная обработка `RetryAfter` от Telegram при превышении лимитов.
- **Устойчивость к ошибкам фида:** если фид не отвечает 5 проверок подряд,
  бот один раз предупредит вас об этом (а не будет спамить каждый раз).
